# TranslationProvider

The `TranslationProvider` component integrates internationalization (i18n) support for NeetoUI components in your React application using `react-i18next`.
It dynamically loads translation resources and switches languages based on the provided props.

NeetoUI supports multiple languages internally, including English (en), Spanish (es), French (fr), German (de), Dutch (nl), Polish (pl), and Portuguese (pt).

## Usage

To use the `TranslationProvider`, wrap your application or specific components that require multilingual support:

```jsx
import React from "react";
import TranslationProvider from "@bigbinary/neetoui/TranslationProvider";

const App = () => {
  return (
    <TranslationProvider
      language="en" // Specify the selected language
      translationResources={{ en: { translation: { /* English translations */ } } }} // Optional prop for translation resources for selected language
    >
      {/* Your application components */}
      <div>
        <h1>Welcome to My App</h1>
        {/* Other components */}
      </div>
    </TranslationProvider>
  );
};

export default App;
```

The `TranslationProvider` component accepts the following props:

- `language` (string): The language to use.
  >> Note: The `language` prop is optional. If not provided, the default language is set to `en`.

- `translationResources` (object): An object containing translation resources for selected language.
If provided with the `language` prop, the component will use the corresponding translation resources for the given language.

## Using translation files from NeetoUI

NeetoUI supports multiple languages internally, including English (en), Spanish (es), French (fr), German (de), Dutch (nl), Polish (pl), and Portuguese (pt).
You can import the translation files for these languages from `@bigbinary/neetoui/{{language_code}}.json` and integrate them into your existing `react-i18next` setup
to enable multilingual support for NeetoUI components.


### Integration Example:

If your application already uses react-i18next, follow these steps to incorporate NeetoUI translations:

>> Import the translation files (en.json used as an example below) from NeetoUI:

```jsx
import React from "react";
import { initReactI18next } from "react-i18next";
import i18n from "i18next";
import en from "@bigbinary/neetoui/translations/en.json"; // Spanish translations from NeetoUI

// Initialize i18next
i18n
  .use(initReactI18next) // passes i18n down to react-i18next
  .init({
    resources: {
      en: {
        translation: {
          // Your application's existing translations
          ...en // Merge NeetoUI translations with your existing translations
        }
      },
      // Add other languages as needed
    },
    // Other i18next configuration options
  });

const App = () => {
  return (
    <div>
      <h1>Welcome to My App</h1>
      {/* Your application components */}
    </div>
  );
};

export default App;

```
### Using NeetoUI Components:

After initialization, NeetoUI components will use the imported translations automatically based on the user's language preference.
