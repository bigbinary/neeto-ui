# TranslationProvider

The `TranslationProvider` component integrates internationalization (i18n) support into your React application using `react-i18next`.
It dynamically loads translation resources and switches languages based on the provided props.

## Usage

To use the `TranslationProvider`, wrap your application or specific components that require multilingual support:

```jsx
import React from 'react';
import TranslationProvider from '@bigbinay/neetoui/TranslationProvider';

const App = () => {
  return (
    <TranslationProvider
      language="en" // Specify the default language
      translationResources={{ en: { translation: { /* English translations */ } } }} // Provide translation resources
    >
      {/* Your application components */}
      <div>
        <h1>Welcome to My App</h1>
        {/* Other components */}
      </div>
    </TranslationProvider>
  );
};

export default App;
```

The `TranslationProvider` component accepts the following props:

- `language` (string): The default language to use.
>> Note: The `language` prop is optional. If not provided, the default language is set to `en`.
>> In addition to English, internal support is provided for Spanish (language code "es") and French (language code "fr") languages.
- `translationResources` (object): An object containing translation resources for each language.


## Initialization with NeetoUI Components

If your application does not have `react-i18next` initialized, `TranslationProvider` initializes default translations for NeetoUI components automatically.

## Using Existing Translation Files

If your host application already has `react-i18next` initialized, you can use the translation files exported from
NeetoUI to initialize the translations for NeetoUI components. Ensure to import and integrate these files into your existing i18n setup.

This setup ensures seamless integration of multilingual support with NeetoUI components in your React applications.
